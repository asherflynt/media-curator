Free space

Safety

The new-release window is a hard exclusion, not a weight: a recent release is one "you may not have got to yet", and a 90 GB remux scores high enough on impact that any survivable penalty would still reach the top of the queue. At 24 months this protects ~133 titles and still leaves ~21.7 TB reclaimable.

Score weights

Ranking uses these two signals only. Raising age promotes canonical classics — The Godfather and Schindler's List rank above Transformers on that axis alone — so check the Candidates page after changing weights, and use the exclusion tag ({{ settings.exclusion_tag }}) for anything you'd regret losing.

Demotion

Tier fallback depth is how many tiers below the profile's top quality a demotion may settle for when the top tier has no release. At 0 a title with no Bluray-1080p available is skipped on every pass forever — it never demotes and never frees anything. The ladder is the profile's own allowed list, walked downward, and it stops above the file's current tier, so a fallback is always still a demotion.

Blocked imports

Radarr checks for an upgrade again at import time, comparing quality weights rather than your profile order, so a completed Bluray-1080p download is rejected over an existing Remux-2160p no matter which profile the movie is on. The download then parks in the queue waiting for someone to click through the manual-import dialog. With this on, the curator does that itself: the existing file goes to Radarr's Recycle Bin (still recoverable, and this is what actually frees the space), then the replacement is manually imported, which bypasses the upgrade check by design. Only downloads blocked on that specific rejection, for movies on a curator-managed profile, are touched.

Off by default, the sweep refuses to run unless Radarr has a Recycle Bin, so a demotion stays recoverable. Turn this on to free space the instant the old file is removed rather than after Recycle Bin retention expires — the tradeoff is that demotions become irreversible: the recovery plan is re-downloading the title.

Path mapping

Translates Radarr's file paths to the read-only mount for free-space and the orphan walk. The mount point itself (/media) is fixed by the container's volume. Check the dashboard's path-mapping banner after changing these.

Archive-HD track (kids movies)

Separate from the free-space loop: you choose titles by assigning them to this profile in Radarr; media-curator just runs the search+grab Radarr won't do automatically after a profile change. Runs on the same interval as the loop.
